# Codex 任务提示

## 0. 长程目标提示

使用 `docs/codex_target_prompt.md` 作为分阶段目标入口。

## 1. 仓库启动模板

1. 读取 `AGENTS.md`、`docs/architecture.md`、`TASKS.md`
2. 提出最小实现计划（优先处理当前阶段任务）
3. 明确缺失源/许可证/签名问题并列出阻塞项
4. 明确“已完成/未完成”与风险后再进入实现

## 2. 单任务实现模板

实现任务 `<TASK_ID>` 时，遵循：

- 仅修改必要文件
- 先补齐文档契约，再补齐实现
- 对应运行最小相关校验
- 返回：变更文件 / 命令 / 输出 / 假设 / 风险

### 示例结构

```text
- 本任务目标
- 实施范围
- 变更文件
- 变更验证
- 剩余风险
- 下一步建议
```

## 3. 评审流程

1. 逐项检查语义安全、数据来源、图谱约束、可复现性
2. 不做无关模块修改
3. 不在报告中做未经验证的性能表述

## 4. 修复与复验循环

- 只修复已确认的缺陷
- 先回归回归测试，再补充新校验
- 所有改动完成后给出可交付状态与残留风险

## 5. 发布与验收

- 明确可复现命令与证据清单（`stats.json`、`report_inputs`、截图）
- 对外发布前清点 `deliverables` 与 `package-manifest`
